## 2. 画面遷移

### 2.1 認証フロー
1. ログイン
2. ダッシュボードへ遷移

### 2.2 レシート OCR フロー
1. ダッシュボードまたは取引画面からアップロード
2. OCR ジョブ待機（ポーリング）
3. 確認画面で抽出結果を確認、修正
4. 確定すると取引へ反映
5. 失敗時は手動入力または再アップロード

#### OCR ポーリング仕様
- `GET /receipt-ocr-jobs/:id` を 2 秒ごとに叩く
- `status` が `success` になったら確認画面へ自動遷移する
- `status` が `failed` になったらエラー表示し、手動入力 / 再アップロードへ誘導する
- 30 秒以内に `success` にならない場合はタイムアウトとしてエラー表示する

### 2.3 サブスク OCR フロー
1. サブスク画面からスクリーンショットをアップロード
2. OCR ジョブ待機（ポーリング）
3. 確認画面で抽出結果を確認、修正
4. 確定するとサブスクへ反映
5. 失敗時は手動入力または再アップロード

#### OCR ポーリング仕様
- レシート OCR と同じポーリング仕様を適用する（2 秒間隔、30 秒タイムアウト）

## 3. ダッシュボード

### 3.1 表示項目
- 今月の支出合計
- 先月比差額
- 先月比増減率
- サブスク月額換算合計
- 直近の支払い予定
- 月表示カレンダー

### 3.2 カレンダー
- 初期表示は月
- 日、週、月、年の切替を用意する
- 支出、収入を切り替えられる
- セルクリックで対象期間の取引一覧へ遷移する

## 4. 取引画面

### 4.1 一覧
- 表形式またはカード形式で表示する
- フィルタ、検索、ソートを提供する
- カテゴリ未設定も表示できる

### 4.2 登録フォーム
- 種別
- 金額
- 日付
- 店名
- カテゴリ
- 支払い方法
- メモ

### 4.3 OCR 確認画面
- 元画像は表示しない
- 抽出された店名、日付、金額、全文テキストを表示する
- 確定後に全文テキストは削除する

## 7. 画面状態定義

各画面で発生しうる状態と表示方針を定義する。

### 7.1 ダッシュボード

| 状態 | 表示 |
| --- | --- |
| ローディング中 | 各カードをスケルトン表示 |
| データあり | 通常表示 |
| 取引が 0 件 | 「まだ取引がありません」+ 取引登録ボタン |
| API エラー | 「データの取得に失敗しました」+ 再試行ボタン |

### 7.2 OCR 待機画面

| 状態 | 表示 |
| --- | --- |
| 処理中 | スピナー + 「読み取り中...」 |
| success | 確認画面へ自動遷移 |
| failed | 「読み取りに失敗しました」+ 手動入力 / 再アップロードボタン |
| タイムアウト | 「読み取りがタイムアウトしました」+ 再試行ボタン |

### 7.3 取引一覧

| 状態 | 表示 |
| --- | --- |
| ローディング中 | スケルトン表示 |
| データあり | 一覧表示 |
| 検索・フィルタ結果が 0 件 | 「条件に一致する取引がありません」 |
| 取引が 0 件（未登録） | 「まだ取引がありません」+ 登録ボタン |
| API エラー | 「データの取得に失敗しました」+ 再試行ボタン |

### 7.4 サブスク一覧

| 状態 | 表示 |
| --- | --- |
| ローディング中 | スケルトン表示 |
| データあり | 一覧表示 |
| サブスクが 0 件（未登録） | 「まだサブスクが登録されていません」+ 登録ボタン |
| API エラー | 「データの取得に失敗しました」+ 再試行ボタン |

### 7.5 通知一覧

| 状態 | 表示 |
| --- | --- |
| ローディング中 | スケルトン表示 |
| 通知あり | 一覧表示（未読を上部に表示） |
| 通知が 0 件 | 「通知はありません」 |

### 7.6 フォーム共通（取引登録・サブスク登録）

| 状態 | 表示 |
| --- | --- |
| 送信中 | 送信ボタンをローディング表示、多重送信を防止 |
| バリデーションエラー | 各フィールド下にエラーメッセージ表示 |
| 送信成功 | 一覧画面へ遷移 |
| API エラー | トースト通知でエラー表示 |

## 8. App Router 構成方針

### 7.1 ルート例
- `/auth/login`
- `/auth/register`
- `/dashboard`
- `/transactions`
- `/transactions/new`
- `/transactions/ocr/:jobId`
- `/subscriptions`
- `/subscriptions/new`
- `/subscriptions/ocr/:jobId`
- `/notifications`
- `/settings`

### 7.2 実装方針
- レイアウトは `app/` 配下で共通化する
- 認証後画面はダッシュボードレイアウト配下へまとめる
- API 呼び出しはサーバープロキシ経由または直接 API クライアントで扱う
