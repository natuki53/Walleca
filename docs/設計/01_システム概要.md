# 01. システム概要

## 1. システム構成

### 1.1 全体像
- クライアントは Next.js App Router による Web アプリとする
- API サーバーは Node.js と TypeScript で実装する
- 永続データは PostgreSQL に保存する
- OCR ジョブ制御と通知スケジュールには Redis を利用する
- OCR エンジンは Tesseract を利用する

### 1.2 構成図

```text
Browser
  -> Next.js Web App
  -> API Gateway / Reverse Proxy
  -> Node.js API Server
      -> PostgreSQL
      -> Redis
      -> OCR Worker
```

## 3. 主要設計方針

### 3.1 Web 優先
- 卒業研究の完成を優先し、初期段階では Web 版に集中する
- API は将来のモバイルアプリ展開を考慮して設計する

### 3.2 OCR データの扱い
- アップロード画像は OCR 実行後に削除する
- OCR の確認前のみ全文テキストを保持する
- ユーザーが確認して確定した後は、構造化データのみを永続化する

### 3.3 個人利用前提
- 1 ユーザーが自身の家計簿を管理する
- 共有、権限制御、グループ機能は MVP の対象外とする

### 3.4 閲覧体験
- ダッシュボード、一覧、カレンダーを中心とする
- カレンダーは日、週、月、年単位の切替を提供する
- 支出、収入の切替表示に対応する

## 4. 主要ユースケース

### 4.1 レシートから支出登録
1. ユーザーが画像をアップロードする
2. OCR ジョブが実行される
3. 抽出結果を確認、修正する
4. 確定すると Transaction が生成される
5. 未確定データと画像は削除される

### 4.2 サブスクのスクリーンショット登録
1. ユーザーが決済画面をアップロードする
2. OCR ジョブが実行される
3. 抽出結果を確認、修正する
4. 確定すると Subscription が生成される
5. 未確定データと画像は削除される

### 4.3 手動登録
- 画像なしで Transaction を追加できる
- 画像なしで Subscription を追加できる

## 5. MVP 範囲

- サブスク管理
- サブスク OCR
- ダッシュボード
- カレンダーと一覧
- 通知設定
