# 05. インフラ運用設計

## 1. 構成
- Next.js Web
- Node.js API
- PostgreSQL
- Redis
- OCR Worker
- Nginx

## 2. Docker Compose 方針

### 2.1 サービス
- `web`
- `api`
- `postgres`
- `redis`
- `ocr-worker`
- `nginx`

### 2.2 役割
- `web`: Next.js の配信
- `api`: 業務 API
- `ocr-worker`: Tesseract を使った OCR 実行
- `postgres`: 永続データ保存
- `redis`: ジョブキュー、キャッシュ
- `nginx`: HTTPS 終端とリバースプロキシ

## 3. 画像ファイル運用

### 3.1 原則
- OCR 用画像は永続保存しない
- 一時ディレクトリへ保存後、OCR 完了後に削除する

### 3.2 一時ファイル削除
- 正常終了時に削除する
- 異常終了時も定期クリーンアップで削除する

## 4. OCR ジョブ運用

### 4.1 ステータス
- `pending`
- `processing`
- `success`
- `failed`
- `confirmed`
- `expired`

### 4.2 自動削除
- 確認されなかったジョブは `expires_at` 到達後に削除する
- 画面離脱時の破棄 API も提供する

## 4.3 定期ジョブ一覧

| ジョブ | スケジュール | 処理内容 |
| --- | --- | --- |
| OCR ジョブ掃除 | 毎時 | 期限切れの `receipt_ocr_jobs`、`subscription_ocr_jobs` を削除する |
| 通知生成 | 毎日 AM 0:00 | ユーザー設定に基づいてサブスク支払通知を生成する |
| サブスク支払日更新 | 毎日 AM 0:00 | `status = active` かつ支払日が過去のサブスクの `next_payment_date` を次周期へ進める |
| バックアップ | 毎日 AM 3:00 | PostgreSQL のデータを `pg_dump` でエクスポートし、7 世代保持する |

## 5. バックアップ

### 5.1 対象
- PostgreSQL の永続データ
- アプリ設定ファイル（`docker-compose.yml`、`.env` 等）

### 5.2 対象外
- OCR 元画像
- 一時 OCR テキスト

### 5.3 バックアップ仕様

| 項目 | 内容 |
| --- | --- |
| 頻度 | 毎日 AM 3:00（cron） |
| 保持世代数 | 直近 7 世代（8 世代目以降は自動削除） |
| 形式 | `pg_dump` による `.sql.gz` |
| 保存先 | ホストマシン上の指定ディレクトリ（例: `/var/backups/walleca/`） |
| 実行方法 | `docker exec postgres pg_dump -U <user> <dbname> \| gzip` |

### 5.4 リストア手順

```bash
# バックアップファイルを解凍して復元する
gunzip -c /var/backups/walleca/backup_YYYYMMDD.sql.gz \
  | docker exec -i postgres psql -U <user> <dbname>
```

### 5.5 crontab 設定例

```cron
0 3 * * * /opt/walleca/scripts/backup.sh >> /var/log/walleca-backup.log 2>&1
```
