# Walleca 使い方ガイド

環境構築・開発サーバー起動・テスト実行・よく使うコマンドをまとめています。

---

## 1. プロジェクト構成

```
Walleca/
├── api/               # バックエンド API（Node.js + Express + Prisma）
├── web/               # フロントエンド（Next.js）
├── docs/              # ドキュメント
└── docker-compose.yml # PostgreSQL・Redis のコンテナ定義
```

---

## 2. 前提条件

| ツール | バージョン |
|--------|-----------|
| Node.js | 20.x 以上 |
| npm | 9.x 以上 |
| Docker / Docker Compose | 最新安定版 |

---

## 3. 環境構築

### 3.1 依存関係インストール

```bash
cd api && npm install && cd ..
cd web && npm install && cd ..
```

### 3.2 Docker で PostgreSQL と Redis を起動

```bash
docker-compose up -d

# 起動確認
docker-compose ps
```

| サービス | URL |
|---------|-----|
| PostgreSQL | `localhost:5432` |
| Redis | `localhost:6379` |

### 3.3 環境変数の設定

#### API（`api/.env`）

```bash
cp api/.env.example api/.env
```

主な設定項目:

| 変数名 | 説明 | 開発時の値 |
|--------|------|-----------|
| `DATABASE_URL` | PostgreSQL 接続文字列 | `postgresql://walleca:walleca_dev_password@localhost:5432/walleca` |
| `REDIS_URL` | Redis 接続 URL | `redis://localhost:6379` |
| `JWT_SECRET` | JWT 署名用シークレット（32 文字以上） | 任意の長い文字列 |
| `JWT_REFRESH_SECRET` | リフレッシュトークン用シークレット | 任意の長い文字列 |
| `PORT` | API サーバーのポート | `3001` |
| `UPLOAD_DIR` | アップロードファイル保存先 | `./uploads` |

> **注意**: Web（Next.js）はデフォルトで `localhost:3000` を使うため、API は `PORT=3001` に設定してください。

#### Web（`web/.env.local`）

```bash
cp web/.env.example web/.env.local
```

主な設定項目:

| 変数名 | 説明 | 開発時の値 |
|--------|------|-----------|
| `API_SERVER_ORIGIN` | API サーバーの URL（Next.js のプロキシが使用） | `http://localhost:3001` |
| `NEXT_PUBLIC_API_URL` | フロントから叩く API ベースパス | `/api` |

### 3.4 データベースのマイグレーション

```bash
cd api
npm run db:migrate
```

初回セットアップはこれだけで完了します。

## 4. 開発サーバーの起動

**ターミナルを 2 つ**（または 3 つ）用意して起動します。

### ターミナル 1: API サーバー

```bash
cd api
npm run dev
```

起動後: `http://localhost:3001/health` でヘルスチェック可能

### ターミナル 2: Web サーバー

```bash
cd web
npm run dev
```

起動後: `http://localhost:3000` でアクセス

### ターミナル 3（任意）: ワーカー

レシート OCR・通知などのバックグラウンド処理を行うワーカーを起動します。

```bash
cd api

# OCR ワーカー + 通知ワーカーを両方起動
npm run worker

# OCR ワーカーのみ
npm run worker:ocr

# 通知ワーカーのみ
npm run worker:notification
```

> OCR アップロード機能を使う場合はワーカーの起動が必要です。Redis が起動している必要があります。

## 5. アプリの機能一覧

| 画面 | URL | 説明 |
|------|-----|------|
| ダッシュボード | `/dashboard` | 今月の支出サマリー・サブスク合計・カレンダー概要 |
| レシート | `/receipts` | レシート画像をアップロードして OCR 読み取り |
| OCR 確認（取引） | `/transactions/ocr/[jobId]` | OCR 結果を確認・修正して取引として登録 |
| 取引一覧 | `/transactions` | 取引の追加・編集・削除 |
| サブスク | `/subscriptions` | サブスクの管理・追加・編集 |
| カレンダー | `/calendar` | 月別の支出カレンダー |
| 通知 | `/notifications` | サブスク支払い前通知の確認 |
| 検索・エクスポート | `/search` | キーワード検索・CSV エクスポート |
| 設定 | `/settings` | プロフィール・パスワード変更 |

## 6. よく使うコマンド

### API（`api/`）

| コマンド | 説明 |
|----------|------|
| `npm run dev` | 開発サーバー起動（ファイル変更で自動リロード） |
| `npm run build` | TypeScript をビルド（`dist/` に出力） |
| `npm start` | ビルド済みを実行（本番用） |
| `npm run worker` | 全ワーカー起動 |
| `npm run worker:ocr` | OCR ワーカーのみ起動 |
| `npm run worker:notification` | 通知ワーカーのみ起動 |
| `npm run db:migrate` | Prisma マイグレーション実行 |
| `npm run db:push` | スキーマを DB に直接反映（プロトタイプ向け） |
| `npm run db:studio` | Prisma Studio 起動（DB の GUI 確認） |
| `npm test` | Jest テスト実行 |
| `npm run test:watch` | テストをウォッチモードで実行 |
| `npm run lint` | ESLint でチェック |
| `npm run typecheck` | TypeScript 型チェック |

### Web（`web/`）

| コマンド | 説明 |
|----------|------|
| `npm run dev` | 開発サーバー起動（`http://localhost:3000`） |
| `npm run build` | Next.js の本番ビルド |
| `npm start` | ビルド後の本番モード起動 |
| `npm run lint` | ESLint でチェック |
| `npm run typecheck` | TypeScript 型チェック |

### DB 操作

```bash
# Prisma Studio で DB の中身を GUI 確認
cd api && npm run db:studio

# DB の中身をリセット（全テーブルを空にしてマイグレーションを再実行）
cd api && npx prisma migrate reset

# Docker コンテナを停止
docker-compose down

# Docker コンテナを停止してボリューム（DB データ）ごと削除
docker-compose down -v
```

## 7. トラブルシューティング

### ログイン・API に接続できない（502 エラー）

- API サーバーが起動しているか確認: `curl http://localhost:3001/health`
- `web/.env.local` の `API_SERVER_ORIGIN` が `http://localhost:3001` になっているか確認
- Web サーバーを再起動（`.env.local` の変更は再起動後に反映）

### DB に接続できない

- `docker-compose up -d` で PostgreSQL が起動しているか確認
- `api/.env` の `DATABASE_URL` のホスト・ポート・ユーザー・パスワードが正しいか確認

### OCR がタイムアウトする

- 開発環境では API 起動時に OCR ワーカーも自動起動します（`AUTO_START_OCR_WORKER=false` で無効化可能）
- 別プロセスで動かしたい場合はワーカーが起動しているか確認（`npm run worker` または `npm run worker:ocr`）
- Redis が起動しているか確認: `docker-compose ps`

### マイグレーションエラー

```bash
cd api
npx prisma migrate reset   # DB をリセットして再マイグレーション
```

### ポートが競合している

- API と Web が同じポートを使っている場合は `api/.env` の `PORT` を `3001` に変更してください
